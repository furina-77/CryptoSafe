# CryptoSafe · 轻量级密码安全服务平台

> 一个基于 Web 技术实现的密码算法工程化演示平台，覆盖对称加密、非对称加密、国密算法、数字签名、密钥管理等核心模块。

---

## 项目简介

CryptoSafe 是一个完整的**密码技术工程化落地**项目，面向以下应用场景：
- 密码算法的工程化实现与演示
- 国密算法体系（GM/T 标准）的实际应用
- 密钥全生命周期管理（KMS）
- 密码服务平台原型设计

## 功能模块

### 1. 对称加密 · AES
| 特性 | 说明 |
|------|------|
| 密钥长度 | AES-128 / AES-256 |
| 工作模式 | CBC（推荐）/ ECB / CTR |
| 填充方式 | PKCS#7 |
| 输入输出 | UTF-8 明文 → Base64 密文 |
| 密钥格式 | Hex / Base64，支持自动生成 |

**工程价值：** CBC 模式配合随机 IV 防重放攻击；CTR 模式支持并行处理，适合大文件流加密。

---

### 2. 非对称加密 · RSA
| 特性 | 说明 |
|------|------|
| 密钥长度 | RSA-2048（演示支持 1024） |
| 填充方式 | OAEP + SHA-256（安全）|
| 密钥格式 | PEM 标准格式 |
| 适用场景 | 密钥协商、短数据保护 |

**注意：** RSA 不适合加密大数据，实际工程中常用"混合加密"（RSA 加密 AES 密钥，AES 加密数据体）。

---

### 3. 国密算法 · SM2 / SM4 / SM3

#### SM4（对称加密）
- 分组大小：128 位（与 AES 相同）
- 密钥长度：128 位
- 模式：CBC / ECB
- 符合标准：GB/T 32907-2016

#### SM2（非对称加密/签名）
- 基于 256 位椭圆曲线（比 RSA-2048 更高效）
- 支持加密和数字签名
- 符合标准：GM/T 0003-2012

#### SM3（哈希摘要）
- 输出长度：256 位
- 压缩函数独立于 SHA-256
- 符合标准：GM/T 0004-2012

**工程意义：** 金融、政务、能源等领域的密码合规要求强制使用国密算法，CryptoSafe 完整实现了 SM 系列算法链路。

---

### 4. 数字签名 · RSA-SHA256 / SM2

**签名流程：**
```
消息 → SHA-256/SM3 哈希 → 私钥签名 → Base64/Hex 签名值
```

**验签流程：**
```
签名值 → 公钥解密 → 比对消息哈希 → 验证结果
```

**安全属性：**
- **不可否认性：** 只有持有私钥的一方能产生有效签名
- **完整性：** 任何篡改都会导致哈希值不匹配
- **身份认证：** 配合证书体系实现 PKI

---

### 5. 密钥管理系统 · KMS

密钥全生命周期管理，符合 NIST SP 800-57 密钥管理规范：

**密钥状态机：**
```
[生成] → ACTIVE → DISABLED → [删除]
                ↓
             EXPIRED（自动）
```

**管理功能：**
- ✅ 密钥生成（AES-256/128、RSA-2048、SM2、SM4）
- ✅ 密钥元数据存储与查询
- ✅ 状态管理（启用/禁用/过期自动检测）
- ✅ 操作审计日志（每次操作有记录）
- ✅ 密钥元数据导出（JSON 格式）
- ✅ 过滤搜索（按类型/状态）

---

### 6. 哈希摘要 · 多算法对比

| 算法 | 输出长度 | 安全性 | 推荐用途 |
|------|---------|--------|---------|
| MD5  | 128 位 | ❌ 已攻破 | 仅校验和（非安全场景）|
| SHA-1 | 160 位 | ⚠️ 弱 | 不推荐新系统使用 |
| SHA-256 | 256 位 | ✅ 安全 | 通用哈希、TLS |
| SHA-512 | 512 位 | ✅ 安全 | 密码存储前预处理 |
| SM3 | 256 位 | ✅ 安全 | 国密体系哈希 |

**HMAC：** 基于密钥的消息认证码，防止哈希值被伪造，用于 API 签名和消息完整性验证。

## 快速开始

1. 下载项目文件（`index.html`, `style.css`, `crypto.js`, `keymgr.js`, `app.js`）
2. 用浏览器直接打开 `index.html`（推荐 Chrome/Edge）
3. 确保网络连接以加载 CDN 密码学库

**注意：** 本项目为演示用途，私钥材料存储在 localStorage，生产环境应使用 HSM（硬件安全模块）或安全的服务端密钥存储。

---

## 安全说明

- RSA 密钥生成在浏览器端执行，演示模式使用 512 位加速（实际部署应使用 2048 位）
- SM4/AES 密钥以明文存储在 localStorage（仅演示）
- 生产级 KMS 应基于 PKCS#11 接口对接硬件安全模块（HSM）
- 所有密码操作在客户端本地完成，不上传任何数据
